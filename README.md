# 📖 神工系统 V3.1 - 自然语言与交互使用指南

神工系统 V3.1 是深度集成于 OpenClaw 的智能化连载写作引擎。在日常使用中，你**不需要输入任何代码或命令行**。你只需要像和人类助手聊天一样，通过自然语言向 OpenClaw 下达指令，底层的所有脚本运行、数据库读写、Agent 调度均由 OpenClaw 自动代劳。

---

## 一、 首次部署与安装

在 OpenClaw 挂载本项目文件夹后，你只需要对 AI 发送一句话即可完成环境搭建。

🗣️ **请对 OpenClaw 发送以下指令：**
> “请帮我初始化并安装神工系统，运行目录下的安装脚本。”

**AI 的后台动作：** 
OpenClaw 会自动检测你的系统环境（Windows/Mac/Linux），自动运行对应的 `install.bat` 或 `install.sh`，帮你装好依赖包并建好 SQLite 数据库。完成后，AI 会在对话中向你汇报“安装就绪”。

---

## 二、 界面交互：启动控制台

神工系统提供了一个直观的 Web 面板，用于让你修改核心大纲、监控角色数值和查看生成日志。

🗣️ **请对 OpenClaw 发送以下指令：**
> “帮我打开神工系统的可视化控制台。”

**AI 的后台动作：** 
OpenClaw 会在后台启动 Streamlit 服务，并在聊天窗口发给你一个本地访问链接（通常是 `http://localhost:8501`）。

💻 **在界面中的操作（鼠标交互）：**
1. **剧本编辑中心**：在文本框内直接填写你的故事一句话大纲（Logline），并写下题材禁忌（如：不许出现修仙元素），点击保存，系统会自动将其固化为世界观约束。
2. **角色状态监控**：在下拉菜单选择你的主角，直接在输入框里修改他的“健康值”或“精神状态”，点击保存，系统会自动原子写入并同步至数据库。
3. **监控面板**：当你让 AI 生成小说时，可以在这里实时刷新，偷看 Writer 和 Critic 两个底层 Agent 的“工作汇报”和“吵架（审核打回）记录”。

---

## 三、 日常运作：自然语言驱动创作

在日常的连载更新中，你完全可以通过自然语言来指挥神工系统，就像在指挥你的专属编辑部。

### 1. 单章推进与生成
当你想要开始写下一章时：

🗣️ **用户输入：**
> “根据当前的世界观设定和前文摘要，帮我生成最新的一章。完成后把内容发给我看看。”

**AI 响应流程：** 
OpenClaw 会自动唤醒 `Orchestrator`（总调度器），依次让 Planner 做规划、Writer 写正文、Critic 做审查，最后把通过质量校验的终稿以人类可读的方式发送在聊天窗口里。

### 2. 动态插手：埋下伏笔
如果你突然有了一个好点子，想要在后续剧情中回收，无需去修改数据库，直接告诉 AI：

🗣️ **用户输入：**
> “在系统的伏笔追踪表里帮我记一条：主角今天在旧城区捡到了一枚带血的怀表。让后面的剧情找机会回收这个线索。”

**AI 响应流程：** 
OpenClaw 会自动调用 `foreshadowing_manager.py` 将这个线索写入 SQLite 的伏笔库。在接下来的生成中，Planner 会自动看到这条待回收的记录并安排相关剧情。

### 3. 世界关系与好感度调整
如果你觉得当前剧情走向需要一点外部干预：

🗣️ **用户输入：**
> “帮我更新一下系统里的角色关系，把主角和财阀巡逻队长的敌对度拉满，信任度降到最低。”

**AI 响应流程：** 
OpenClaw 自动调用 `graph_manager.py` 操作数据库，更新图谱关系。下一章生成时，Writer 会敏锐地捕捉到这两人关系恶化，从而写出相应的冲突戏码。

---

## 四、 终极解放：全自动连载托管 (Cron 调度)

如果你想彻底放手，让神工系统每天自动为你“打工”更新小说，你可以让 OpenClaw 利用内置的定时任务系统托管项目。

🗣️ **用户输入：**
> “请帮我设置一个定时任务：从今天开始，每天晚上 8 点自动运行神工系统生成新的一章，并在完成后总结这章的剧情摘要发给我。”

**AI 响应流程：** 
OpenClaw 会自动创建一个每天运行的计划任务。每天晚上 8 点，它会在后台静默启动 `Orchestrator`，走完全套的生成、审核、记忆录入流程，然后主动在这个聊天窗口给你发消息：“老板，最新一章写完了，本章讲了……”

---

**💡 使用提示总结：**
有了 OpenClaw，**系统代码只是底层工具，语言才是你的界面**。遇到任何需要查看进度、修改设定、推进剧情的需求，只需直接用大白话向系统下达命令即可。
